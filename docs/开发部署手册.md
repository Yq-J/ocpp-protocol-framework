

## Redis 会话注册（高负载/集群推荐）

### 目标
- 在高并发场景下，将会话“连接对象”与“全局归属索引”解耦：
  - 本地 JVM 仅保存 `chargePointId -> WebSocket连接`，保证主动下发最低延迟。
  - Redis 保存 `chargePointId -> nodeId`，保证多节点快速识别连接归属。

### 关键配置
```yaml
ocpp:
  redis-session-registry-enabled: true
  redis-session-registry-ttl-seconds: 180
  node-id: ${HOSTNAME:ocpp-node-01}
  redis-global-guard-enabled: true
  max-inbound-messages-per-second: 12000
  inbound-worker-threads: 32
  inbound-queue-capacity: 300000
```

### 部署建议
1. Redis 建议采用哨兵或集群模式，开启持久化和慢查询告警。
2. `node-id` 必须全局唯一，建议使用容器实例名。
3. TTL 建议设置为心跳周期 2~3 倍，避免临时抖动导致误清理。
4. 结合 Nginx/SLB 四层负载均衡，WebSocket 保持长连接与合理超时。

### 验证步骤
1. 启动 Redis 与 demo 服务。
2. 连接充电桩后执行：`redis-cli keys ocpp:session:cp:*`，确认有注册键。
3. 断开连接后确认键被删除（或 TTL 过期后自动消失）。
4. 压测期间观察应用 CPU、队列长度、Redis QPS 与延迟。
